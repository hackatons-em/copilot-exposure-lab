# Procurement and Security Packet

## Include
1. Product overview.
2. Architecture.
3. Data flow.
4. Microsoft permissions.
5. Data stored/not stored.
6. Retention/deletion.
7. Deployment options.
8. Security controls.
9. Incident contact.
10. Pilot scope.

## Key answers
Do you train on customer data? No.
Do you need write access? No for MVP/pilot.
Can we deploy in our Azure tenant? Planned customer-cloud mode.
Can we delete all data? Yes.
Do you replace Purview/Defender? No.

## Pilot constraints
- Read-only.
- Narrow scope.
- Metadata-only default.
- No automated remediation.
- Customer approves exports.
