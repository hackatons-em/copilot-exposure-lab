# Report Specification

## Report sections
1. Executive summary.
2. Scope and methodology.
3. Findings by severity.
4. Critical finding details.
5. Evidence chains.
6. Remediation plan.
7. Proof-of-fix.
8. Limitations.
9. Data handling appendix.
10. Permissions requested.

## Each finding must include
- Finding ID.
- Title.
- Severity and score.
- Scenario.
- Affected resource.
- Affected users/groups.
- Evidence chain.
- Business impact.
- Remediation.
- Owner/status.

## Report quality bar
The report must be useful without a sales call. A CISO should understand top risks. A Microsoft admin should know what to fix. A security architect should trust the evidence.
