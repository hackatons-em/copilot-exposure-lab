# Kill and Build Criteria

## Build if at least 4 are true in 30 days
- 10+ qualified people confirm active concern.
- 5+ have active Copilot rollout/security review.
- 3+ say existing Microsoft tools do not give the proof/workflow they need.
- 3+ say sample report is useful internally.
- 2+ discuss paid pilot seriously.
- 1+ design partner next step.

## Kill if
- 20 qualified people say Microsoft solves it well enough.
- No active rollout blocker.
- No budget owner.
- Feedback is only "interesting."
- Buyers refuse metadata-only or customer-cloud startup involvement.
- $10k pilot feels expensive to all qualified buyers.
- Demo cannot show a real-feeling exposure path.

## Narrow if
- Buyers care only about SharePoint oversharing.
- Buyers care only about Copilot Studio agents.
- Buyers care only about customer-cloud deployment.

## Rule
A meeting is not validation. A compliment is not validation. A named buyer with budget and timeline is validation.
