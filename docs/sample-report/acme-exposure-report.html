4. Critical & High Finding Detail

fnd-c1be79bf — Sensitive file shared through an organization-wide link Critical 91/100

2026_salary_plan.xlsx is reachable through an organization-wide sharing link.

Business impact: Any of 7 users — and anyone a link is forwarded to — can open 2026_salary_plan.xlsx. Copilot can surface it to all of them.

Exposure path: Bob Novak → Everyone Except External Users → organization-wide link → 2026_salary_plan.xlsx

Evidence chain:

  • (link) Sharing link scope = organization-wide (read) [link:pg-issue1-salary-orgwide]
  • (permission) Grants read access to Everyone Except External Users [permission:pg-issue1-salary-orgwide]
  • (membership) Reachable by 7 internal user(s) [group:g-everyone]
  • (sensitivity) Sensitive content: salary, payroll, compensation, hr [file:f-salary]

Remediation (SharePoint sharing policy + Purview sensitivity label): Remove the organization-wide link and restrict to the intended audience

  1. Delete the organization-wide / 'people in your org' sharing link on the file.
  2. Grant access only to the specific team that needs it (e.g. HR Leadership).
  3. Apply a sensitivity label (e.g. Confidential) to enforce handling downstream.
  4. Set the SharePoint site/library default sharing to 'Specific people'.

fnd-1cb3d835 — Sensitive file inherits broad read from its parent site High 75/100

2026_acquisition_strategy.pptx inherits broad read from Board Room.

Business impact: 2026_acquisition_strategy.pptx is readable by the whole organization through inherited site permissions — likely unintended for board-level content.

Exposure path: Bob Novak → Everyone Except External Users → Board Room → 2026_acquisition_strategy.pptx

Evidence chain:

  • (permission) Site Board Room grants read to Everyone Except External Users [site:s-board]
  • (permission) 2026_acquisition_strategy.pptx inherits this read grant (inheritance not broken) [permission:pg-issue3-board-everyone]
  • (membership) Everyone Except External Users effectively covers the whole organization [group:g-everyone]
  • (sensitivity) Sensitive content: acquisition, board, confidential, executive [file:f-acq]

Remediation (SharePoint unique permissions): Break inheritance and remove whole-org read on sensitive content

  1. Stop inheriting permissions on the sensitive library/file.
  2. Remove the 'Everyone except external users' read grant inherited from the site.
  3. Grant access explicitly to the board/executive audience only.

Helpdesk Assistant can mail.send and external.connector — a risky action for an agent over company data.

Business impact: Helpdesk Assistant could send sensitive summaries to external recipients on a user's behalf. Combined with broad data access, that is an exfiltration path.

Exposure path: n/a

Evidence chain:

  • (agent-config) Agent actions enabled: mail.send, external.connector [agent:a-helpdesk]
  • (action) Connectors: Office365Outlook, HTTPWithAzureAD [action:a-helpdesk]
  • (ownership) Authentication mode: maker; owner: Tomas Berg [user:u-tomas]

Remediation (Copilot Studio governance + Power Platform DLP policy): Restrict the agent's risky actions and connectors

  1. Remove or gate the agent's send-mail / external connector actions.
  2. Require approval / DLP policy for outbound actions from agents.
  3. Scope the agent's data access to only what it needs.
